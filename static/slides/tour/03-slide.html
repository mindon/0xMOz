② 寫 slide 內容頁 — slides/*.html

每個 slide 內容頁就是一個完整 HTML 文件。可被 index 串聯，也可被瀏覽器直接打開。

❶ 最小骨架

<!DOCTYPE html>
<html lang="zh_CN">
  <head>
    <meta charset="utf-8">
    <link href="../assets/present.css" rel="stylesheet">
    <script src="../present.n8e91kk6.js"></script>
  </head>
  <body>
    <a href="../tour.html" class="home"></a>
    <slide class="slide-center" describe="我的內容頁">
      <div>
        <h1>Hello ∅M〇ᶻ</h1>
      </div>
    </slide>
  </body>
</html>

注意：index.html 的 <slide> 屬性（如 topic、data-pop）會與此處 <slide> 屬性合併（class 累加）。 因此可在內容頁聲明 popover/lazy 等獨立可用的配置。

❷ importmap 模塊映射

每個 slide 內容頁可以自帶 <script type="importmap">，present.n8e91kk6.js 會在加載時將其合併到主文檔的 importmap 中：(⚠️注意：直接放置一個importmap合併版本在index頁面head中，以防止出現瀏覽器兼容問題) 

<head>
  ...
  <script type="importmap">
  {
    "imports": {
      "@lit/reactive-element/": "https://cdn.jsdelivr.net/npm/@lit/reactive-element@2.1.1/",
      "three":           "https://cdn.jsdelivr.net/npm/three@0.180.0/build/three.module.min.js",
      "three/addons/":   "https://cdn.jsdelivr.net/npm/three@0.180.0/examples/jsm/"
    }
  }
  </script>
</head>
  • imports / scopes / integrity 三類字段都會被遞歸合併
  • 多個 slide 重複聲明同一鍵時，後加載者覆蓋前者
  • 該 slide 獨立打開時，importmap 同樣生效（瀏覽器原生）

➌ data-lazy 懶加載 + lazies.slide

滾動到該頁時才執行的初始化邏輯。流程：
步驟1： <slide data-lazy="my-key"> 聲明；
步驟2： 在頁內 <script type="module"> 中 lazies.slide("my-key", fn) 註冊；
步驟3： 框架在頁面進入視口後調用 fn(slide)。 

<slide class="slide-center" data-lazy="chart">
  <canvas id="my-chart"></canvas>

  <script type="module">
    const { lazies, lazy, q$ } = window._0xMOz;

    lazies.slide("chart", (slide, rel = "./") => {
      const list = lazy([
        "https://cdn.example.com/chart.css",
        "https://cdn.example.com/chart.js?module",
      ]);
      list[1].onload = () => {
        // 初始化（資源加載完成後）
        const ctx = q$("#my-chart");
        // ... 繪製圖表
        slide.removeAttribute("data-lazy"); // 顯式釋放 loading 遮罩
      };
    });
  </script>
</slide>

❝ fn 收到第二參 rel：在 index 模式下為 "slides/"，獨立打開時為 "./"， 可用來統一拼接相對路徑。
❝ 若 fn(slide) 返回 false，框架會保留 data-lazy 以便重試。

➍ window._0xMOz API

const { q$, q$$, lazy, lazies } = window._0xMOz;

// q$(selector, scopeOrCallback?) — 查單個元素
const el = q$("#hero");
q$(".btn", (dom) => dom.addEventListener("click", onClick));

// q$$(selector, callback?) — 查所有元素
q$$("a", (a) => a.target = "_blank");

// lazy([...]) — 動態插入 <script>/<link>，按 URL 內容緩存
const [css, js] = lazy([
  "https://cdn.../style.css",
  "https://cdn.../app.js?module",
]);
js.onload = () => { /* ... */ };

// lazies.slide(key, fn) / lazies.slide({ k1: fn1, k2: fn2 })
lazies.slide("hero", (slide, rel) => { /* init */ });

➎ 在線/離線分支：web-or-not

<slide web-or-not> 配合 URL 參數 ?web 與 <meta name="online-detector"> 啟用網絡探測：

  • 探測在線：保留 .web-yes 元素，移除 .web-no
  • 探測離線：相反；元素若帶 .stop()/.abort() 會被先調用以清理

➏ 小貼士

  • 內容頁中 src="./xxx.png" 的相對路徑會被自動改寫為 slides/xxx.png
  • 內容頁中的 <script>（含 module）會被重新創建以保證執行
  • 每個 slide 高度為 100vh；內容過多請自行 overflow-y: auto
